## 关键节点处理规则

### 节点A：参数「未提及」处理

当提取到的参数标记为「未提及」时：

1. **优先级判定**
   - 高优先级：影响施工安全的参数（施工限压、封隔器承压等）
   - 中优先级：影响工艺效果的参数（砂比、液量等）
   - 低优先级：参考性质的参数（焖井制度等）

2. **搜索策略**
   - 高优先级 → 自动触发搜索，无需用户确认
   - 中优先级 → 询问用户是否需要搜索
   - 低优先级 → 可选，用户明确要求才搜索

3. **结果融合**
   - 搜索到的数据补充到参数汇总表
   - 注明数据来源和时间

### 节点B：热搜热点关联

每次分析自动抓取当前热搜：

1. **关键词提取** — 从热搜标题中提取与油气工程相关的关键词
2. **相关性评分** — 评估热搜与当前分析内容的相关度
3. **关联输出** — 在「第四部分：行业热点关联」中输出

示例：
```
🔥 当前行业热点关联
├─ 相关热搜：#页岩气压裂技术突破#（热度 987万）
│  └─ 关联内容：最新滑溜水体系应用案例，可参考优化本井液量设计
├─ 相关热搜：#深层油气开发新装备#（热度 654万）
│  └─ 关联内容：深层压裂工具选型建议
└─ 趋势提示：当前行业关注"体积压裂2.0"技术路线
```

### 节点C：风险等级判定

| 等级 | 判定标准 | 输出颜色 |
|------|----------|----------|
| 🔴 高 | 可能导致施工事故或工程失效 | 红色标注 |
| 🟡 中 | 可能影响施工效率或增加成本 | 黄色标注 |
| 🟢 低 | 不影响施工但可进一步优化 | 绿色标注 |

## 输出质量标准

| 检查项 | 标准 |
|--------|------|
| 参数完整性 | 四类核心参数全部覆盖 |
| 原文准确性 | 禁止外推，数据必须可溯源 |
| 建议可落地性 | 每条建议必须有操作步骤 |
| 风险标注 | 每个风险点必须有等级 |
| 热点关联 | 每次分析必须包含热搜关联 |
