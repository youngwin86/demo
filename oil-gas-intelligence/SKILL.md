# 油气工程智能分析助手

> **从数据到决策，只需一次上传**
> 打破信息孤岛，让工程分析跟上行业节奏

---

## 一句话定位

传统工程分析靠经验，新一代靠**全网实时数据 + AI**。

本技能将「热搜抓取」与「压裂设计分析」深度融合，实现**数据驱动**的工程决策闭环。

---

## 解决的核心痛点

### ❌ 传统分析的问题

| 痛点 | 具体表现 |
|------|----------|
| **信息滞后** | 参考标准可能已过期5~10年，方案脱离当前行业水平 |
| **资料分散** | 工具参数查手册，液体系能查论文，标准不统一 |
| **建议主观** | 依赖个人经验，不同工程师结论可能截然不同 |
| **效率低下** | 一份完整分析靠手工需要3~5天 |
| **遗漏热点** | 不知道行业最近在讨论什么问题、什么技术火了 |

### ✅ 本技能带来的改变

| 改变 | 效果 |
|------|------|
| **实时数据融合** | 自动抓取行业最新动态，结合设计文档给出时代化建议 |
| **一键全网搜索** | 自动搜索工具/液体/工艺的最新技术参数 |
| **结构化输出** | 标准模板输出，高/中/低风险等级标注，结论可追溯 |
| **效率提升** | 从3~5天 → 30分钟内完成完整分析 |
| **热点关联** | 分析完成后，自动提示当前行业热点与技术趋势 |

## 工作流程（4步闭环）

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌──────────┐     ┌──────────┐     ┌──────────┐          │
│   │ STEP 1   │────▶│ STEP 2   │────▶│ STEP 3   │          │
│   │ 数据提取 │     │ 全网搜索 │     │ 智能优化 │          │
│   └──────────┘     └──────────┘     └──────────┘          │
│        │                                      │             │
│        │           ┌──────────┐               │             │
│        └─────────▶│ STEP 4   │◀──────────────┘             │
│                    │ 建议输出 │                              │
│                    └──────────┘                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### STEP 1：文档数据提取

**输入**：用户上传 PPTX/DOCX/DOC 压裂设计文档

**处理**：
- 解析文档，提取所有文本内容
- 按四大类结构化参数标准化输出

**输出**：标准参数汇总表

```
✅ 井口与套管参数
✅ 油管与完井管柱
✅ 压裂井下工具
✅ 压裂工艺参数
⚠️  未提及的参数 → 标记「未提及」，触发下一步搜索
```

### STEP 2：全网智能搜索

**触发时机**：分析完成、发现「未提及」参数、或用户要求补充最新数据

**搜索来源**：

| 数据类型 | 来源 | 说明 |
|----------|------|------|
| 热搜动态 | 微博/百度/抖音 | 当前行业热点，提示行业趋势 |
| 工具技术 | 全网搜索 | 封隔器、桥塞、水力锚等最新参数 |
| 液体体系 | 全网搜索 | 压裂液最新配方与性能数据 |
| 工艺标准 | 全网搜索 | 最新施工排量、砂比我们行业规范 |
| 案例分享 | 全网搜索 | 类似井况的成功案例参考 |

**搜索策略**：
- 自动提取文档中标注「未提及」的关键参数
- 针对每个缺失参数发起独立搜索
- 优先返回权威来源（石油工业出版社、行业标准、核心期刊）

### STEP 3：智能优化分析

基于 STEP 1 的参数 + STEP 2 的最新数据，从三个专业视角分析：

#### 🔧 工具工程师视角
- 当前工具与井况匹配性
- 压力安全余量（计算验证）
- 管柱损伤风险

#### 🛠️ 压裂设计工程师视角
- 工艺合理性评估
- 储层改造均匀性
- 加砂风险分析
- 层间窜流风险

#### 💧 压裂液工程师视角
- 液体体系与储层匹配性
- 黏土敏感性防控
- 压后返排风险

### STEP 4：结构化建议输出

**输出格式**（固定模板）：

```
📊 第一部分：核心参数汇总表
   - 完整提取结果
   - 「未提及」项标注来源

⚠️ 第二部分：核心工程问题
   ├─ 工具模块（高/中/低 风险）
   ├─ 压裂模块（高/中/低 风险）
   └─ 液体模块（高/中/低 风险）

💡 第三部分：落地改进建议
   ├─ 建议1（操作方法 + 预期效果）
   ├─ 建议2（操作方法 + 预期效果）
   └─ 建议3（操作方法 + 预期效果）

🔥 第四部分：行业热点关联
   - 当前热搜相关技术趋势
   - 同类井况最新动态
```

## 使用方式

### 方式一：完整分析流程（推荐）

```
用户：帮我分析这份压裂设计方案，结合最新行业动态

助手：
  1. 读取文档 → 提取参数 → 汇总表
  2. 识别「未提及」参数 → 自动搜索
  3. 热搜抓取 → 行业热点关联
  4. 三视角分析 + 风险等级
  5. 输出结构化优化建议
```

### 方式二：快速建议（已知问题）

```
用户：现在压裂液行业有什么新技术？

助手：
  1. 抓取当前热搜/行业热点
  2. 搜索压裂液最新技术
  3. 输出技术对比表 + 建议
```
