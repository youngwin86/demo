# Oil-Gas Intelligence - Engineering Analysis Assistant

> **From data to decision, in one upload**
> Breaking information silos — real-time data meets AI-driven engineering analysis

## Core Problem Solved

Traditional engineering analysis relies on **outdated manuals** and **individual experience**. This skill combines real-time web data with structured engineering analysis to deliver **data-driven decisions**.

## Pain Points vs. This Skill

| ❌ Traditional | ✅ This Skill |
|---------------|--------------|
| Standards 5-10 years old | Real-time web search |
| Scattered information | Unified data pipeline |
| Subjective suggestions | Structured risk levels |
| 3-5 days per analysis | Under 30 minutes |
| No industry context | Hotspot integration |

## Workflow (4-Step Closed Loop)

```
STEP 1: Extract          STEP 2: Search         STEP 3: Optimize         STEP 4: Advise
Document → Parameters → Web Intelligence → Risk Analysis → Recommendations
```

1. **Extract** — Parse PPTX/DOCX/DOC, extract standardized parameters
2. **Search** — Auto-identify missing data, search latest industry standards
3. **Optimize** — Three-perspective analysis (Tool / Fracturing / Fluid)
4. **Advise** — Structured output with risk levels + actionable recommendations

## Quick Start

```bash
#抓取行业热搜
python scripts/fetch_hotspot.py all

#工程数据搜索
python scripts/fetch_engineering_data.py "压裂液体系"

#完整分析（由 AI 助手触发）
用户上传设计文档 → 助手自动执行完整流程
```
